Desafío de Unidades CSS Avanzadas

La altura del header debe ser igual que la altura del dispositivo

Introducción al Experimento

¡Redimensiona tu navegador para observar los cambios!

En este desafío, **modificarás el código CSS** para controlar el tamaño de diferentes elementos HTML. **Tu tarea es añadir las propiedades `width` y `height`** (o solo `width` en el caso de los porcentajes) en las reglas CSS de cada sección, siguiendo las **ÓRDENES EXACTAS** que encontrarás a continuación y en los comentarios del CSS. Al hacerlo, **observarás cómo cada unidad afecta el diseño** al redimensionar la ventana de tu navegador.

Unidades de Píxeles (px) - Grupo 1

Tamaño fijo, sin importar el viewport

Estos elementos **deben mantener siempre sus dimensiones fijas**. Son ideales para elementos que necesitan un tamaño preciso e inmutable, como iconos o botones.

**ÓRDENES EXACTAS:**
- Para **"Caja PX 1"**: Dale un **ancho de 200 píxeles** y un **alto de 60 píxeles**.
- Para **"Caja PX 2"**: Dale un **ancho de 120 píxeles** y un **alto de 35 píxeles**.
- Para **"Caja PX 3"**: Dale un **ancho de 250 píxeles** y un **alto de 70 píxeles**.

Unidades de Porcentaje (%) - Grupo 1

Escalando con el ancho del contenedor padre

Estos elementos **deben escalar en relación con el ancho de su contenedor padre**. Observa cómo se ajustan proporcionalmente cuando redimensiones el contenedor o la ventana del navegador. La altura de estos elementos se ajustará automáticamente por su contenido y padding.

**ÓRDENES EXACTAS:**
- Para **"Caja % A"**: Dale un **ancho del 30%** de su contenedor padre.
- Para **"Caja % B"**: Dale un **ancho del 45%** de su contenedor padre.
- Para **"Caja % C"**: Dale un **ancho del 20%** de su contenedor padre.
